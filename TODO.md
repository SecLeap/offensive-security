# 知识库总待办

## P0：框架与基线

- [ ] 为每个主题建立统一模板：原理、前置条件、风险、授权验证、检测、修复、参考。
- [ ] 定义术语表、缩写表与标签规范。
- [ ] 建立实验记录脱敏规范、授权证明留存规范与数据清理规范。
- [ ] 明确内容分级：公开笔记、内部笔记、受限实验记录。

## P1：基础能力

- [ ] 完成网络、操作系统、HTTP/TLS、数据库、编码与脚本基础笔记。
- [ ] 完成 Linux、Windows、Git、Docker、日志和常用命令行基础笔记。
- [ ] 完成安全测试流程、风险评级、证据链与复测方法笔记。

## P2：专项能力

- [ ] 按目录补齐 Web、API、网络、主机、AD、云、容器、移动和 IoT 的攻击面清单。
- [ ] 为每个漏洞类别补齐成因、验证要点、检测与修复建议。
- [ ] 为每个专项整理安全配置基线与常见误区。

## P3：进阶与场景

- [ ] 建立攻击路径分析与权限边界建模方法。
- [ ] 补齐二进制、逆向、对抗模拟和检测工程的学习路线。
- [ ] 建立互联网暴露面、企业内网、云原生、供应链等场景化案例模板。

## P4：质量与沉淀

- [ ] 为完成主题补充权威参考、版本信息与最后复核日期。
- [ ] 建立知识卡片、复盘模板和季度更新机制。
- [ ] 定期检查过期技术、失效链接和不安全示例。
