# 14｜报告、整改与复测

目标：让技术发现能够被业务、研发、运维和管理者理解并转化为实际改进。

## 主题

- [ ] 管理摘要：风险故事、业务影响、优先级、趋势和资源建议。
- [ ] 技术发现：受影响范围、前置条件、最小化复现、证据和根因。
- [ ] 修复建议：短期缓解、长期治理、所有者、依赖关系和验证标准。
- [ ] 风险例外：接受条件、到期时间、补偿控制与复核机制。
- [ ] 复测：修复确认、回归检查、关闭标准与残余风险说明。
- [ ] 指标与复盘：修复 SLA、重复问题、根因类别和改进计划。

## 达标产物

- [ ] 管理层与技术报告模板。
- [ ] 整改跟踪、风险接受、复测关闭模板。
- [ ] 报告审校清单（准确性、脱敏、可执行性）。
