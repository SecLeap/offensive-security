# 13｜场景化攻防与渗透测试

目标：按真实业务边界组织测试，而不是按孤立漏洞堆砌知识。

## 场景清单

- [ ] 互联网暴露面：官网、门户、API、VPN、邮件与远程运维。
- [ ] 企业内网：办公网、身份域、文件服务、管理网络与分支互联。
- [ ] 云原生：多账号、多租户、Kubernetes、CI/CD 与 SaaS 集成。
- [ ] 软件供应链：源码、依赖、构建机、制品库、签名与发布渠道。
- [ ] 数据与 AI：数据仓库、对象存储、检索增强、模型服务、插件与权限。
- [ ] 移动与物联网：应用、设备、后台、OTA 与第三方平台。
- [ ] 并购/第三方：接入评估、信任边界、共享数据与持续监控。

## 每个场景应包含

- [ ] 业务资产、数据流、角色和信任边界。
- [ ] 授权范围、生产限制、回退方案和联系人。
- [ ] 攻击面假设、验证优先级、检测点和修复路线。
- [ ] 交付物：管理摘要、技术发现、证据、整改建议和复测计划。
