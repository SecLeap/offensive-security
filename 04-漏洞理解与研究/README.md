# 04｜漏洞理解与研究

目标：从根因出发判断风险，在隔离环境中完成最小化验证，并给出可落地的修复建议。

## 主题

- [ ] 漏洞根因分类：输入处理、身份认证、授权、配置、依赖、逻辑与竞态。
- [ ] 漏洞情报：CVE、CWE、厂商公告、补丁差异和版本影响判断。
- [ ] 安全验证原则：最小影响、可逆、限速、隔离、停止条件与证据留存。
- [ ] 代码审计基础：数据流、控制流、信任边界、危险 API 与安全编码模式。
- [ ] 漏洞定级与可利用性评估：可达性、前置条件、影响范围和防护状态。
- [ ] 修复验证：补丁有效性、回归风险、旁路检查和防御纵深。

## 达标产物

- [ ] CWE 到常见修复模式的映射表。
- [ ] 漏洞验证与修复复测模板。
- [ ] 漏洞情报订阅与筛选流程。
